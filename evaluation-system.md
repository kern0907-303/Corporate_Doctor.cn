## 二、计分公式

### 2.1 四大瓶颈分数计算

#### B1｜市场/需求瓶颈

```
B1_base = Q1 市场相关选项数 × 1.5

B1_chronic = Q7 中市场相关选项数 × 1.0
B1_acute = Q8 中市场相关选项数 × 1.5

B1_condition_bonus = IF(Q3 ≤ 3 AND Q4 ≤ 3) THEN 1 ELSE 0
// 决策不慢、执行不差，但还是缺客 → 更确定是市场问题

B1_score = B1_base + B1_chronic + B1_acute + B1_condition_bonus
```

**B1 相关选项**：
- Q1：新客来源不稳、成交率不如预期
- Q7：客户开发一直没有稳定方法
- Q8：业绩/订单明显下滑、重要客户流失或关系恶化

---

#### B2｜管理/决策瓶颈

```
B2_quantitative = (Q3 × 0.35) + (Q4 × 0.30) + (Q6 × 0.25) + (Q5 × 0.10)

B2_chronic = Q7 中管理相关选项数 × 1.0

B2_score = B2_quantitative + B2_chronic
```

**B2 相关选项**：
- Q1：决策常被拖慢、老板负担过重
- Q7：老板是最大瓶颈、管理制度一直建不起来

---

#### B3｜执行/产能瓶颈

```
B3_quantitative = (Q4 × 0.45) + (Q5 × 0.30) + (Q6 × 0.25)

B3_chronic = Q7 中执行相关选项数 × 1.0
B3_acute = Q8 中执行相关选项数 × 1.5

B3_score = B3_quantitative + B3_chronic + B3_acute
```

**B3 相关选项**：
- Q1：团队执行力不一致、人员留不住
- Q7：团队执行力长期不稳定
- Q8：关键人员离职或闹离职、某个专案/产品严重卡关

---

#### B4｜现金流瓶颈

```
B4_subjective = IF(Q1 包含「现金流压力」) THEN 3 ELSE 0
             + IF(Q2 包含「现金流安全感」) THEN 2 ELSE 0

B4_chronic = IF(Q7 包含「现金流一直都偏紧」) THEN 1.5 ELSE 0
B4_acute = IF(Q8 包含「现金流突然吃紧」) THEN 2.0 ELSE 0

B4_score = B4_subjective + B4_chronic + B4_acute
```

---

### 2.2 主瓶颈判定逻辑

```
Step 1｜现金流优先检查
IF B4_score ≥ 4 THEN 主瓶颈 = B4，跳到结束

Step 2｜Boolean 条件检查
B1_valid = (Q1市场选项 ≥ 1) AND (Q3 ≤ 3) AND (Q4 ≤ 3)
B2_valid = (Q3 ≥ 4 AND Q4 ≥ 4) OR (Q6 ≥ 4)
B3_valid = (Q4 ≥ 4) AND (Q3 ≤ 3)
B4_valid = (Q1 包含「现金流压力」) OR (Q2 包含「现金流安全感」)

Step 3｜从有效瓶颈中选最高分
valid_bottlenecks = 所有 Bx_valid = TRUE 的瓶颈
主瓶颈 = valid_bottlenecks 中 score 最高者

Step 4｜平手裁决(依存活优先序)
IF 多个瓶颈同分 THEN
  优先序：B4 > B2 > B3 > B1
  主瓶颈 = 优先序最高者

Step 5｜无有效瓶颈时
IF valid_bottlenecks 为空 THEN
  主瓶颈 = 所有瓶颈中 score 最高者
```

---

### 2.3 灯号判定逻辑

```
计算紧急度总分：

urgency_score = 
  主瓶颈分数
  + (Q8 勾选数 × 1.5)
  + IF(Q8 包含「现金流突然吃紧」) THEN 2 ELSE 0
  + IF(Q6 ≥ 4) THEN 1 ELSE 0

灯号判定：

🔴 红灯条件(任一成立)：
  - urgency_score ≥ 7
  - B4_score ≥ 4
  - Q8 包含「现金流突然吃紧」
  - (Q8 勾选数 ≥ 2) AND (任一急性项目权重 ≥ 1.5)

🟡 黄灯条件(任一成立)：
  - 4 ≤ urgency_score < 7
  - 主瓶颈分数 ≥ 3
  - Q3 ≥ 4 OR Q4 ≥ 4 OR Q5 ≥ 4 OR Q6 ≥ 4
  - Q7 勾选数 ≥ 3

🟢 绿灯条件：
  - 以上都不符合
```

---

### 2.4 问题性质判定

```
acute_count = Q8 勾选数(排除「没有，最近还算稳定」)
chronic_count = Q7 勾选数(排除「以上都没有」)

问题性质判定：

「急性为主」：acute_count ≥ 2 AND chronic_count ≤ 1
「慢性为主」：chronic_count ≥ 3 AND acute_count ≤ 1
「复合型」：acute_count ≥ 1 AND chronic_count ≥ 2
「稳定型」：acute_count ≤ 1 AND chronic_count ≤ 1
```

---

### 2.5 行动能力计分

```
time_score = Q9 分数(1-4)
helper_score = MIN(Q10 分数加总, 3)(0-3，封顶)
confidence_score = Q14 分数 × 2(0-6)

action_ability_score = time_score + helper_score + confidence_score

范围：1-13 分

行动能力分级：
  低(1-4 分)：认知建立，小成功体验
  中(5-8 分)：认知 + 简化行动  
  高(9-13 分)：完整实战计划
```

---

### 2.6 目标类型判定

```
根据 Q12 勾选项目：

IF 勾选任一「止血型」选项 THEN goal_type = "止血"
ELSE IF 勾选任一「修复型」选项 THEN goal_type = "修复"
ELSE IF 勾选任一「成长型」选项 THEN goal_type = "成长"

IF 勾选多个类型 THEN
  优先序：止血 > 修复 > 成长(先解决生存问题)
  但保留次要目标作为计划的第二主轴
```

## 四、7天计划生成逻辑

### 4.1 计划生成矩阵

```
7天计划 = f(主瓶颈, 问题性质, 行动能力, 目标类型, 优势能力, 产业)
```

#### 主轴选择(主瓶颈 × 问题性质)

| 主瓶颈 | 急性为主 | 慢性为主 | 复合型 | 稳定型 |
|--------|---------|---------|--------|--------|
| B1 市场 | 紧急获客 | 建立获客系统 | 止血+建系统 | 优化转换 |
| B2 管理 | 快速决策清理 | 建立决策节奏 | 清理+建节奏 | 决策优化 |
| B3 执行 | 危机任务处理 | 建立执行SOP | 处理+建SOP | 效率提升 |
| B4 现金流 | 紧急资金调度 | 现金流体质改善 | 调度+改体质 | 现金流优化 |

#### 强度调整(行动能力)

| 行动能力 | 每日任务量 | 任务性质 | 目标设定 |
|---------|-----------|---------|---------|
| 低(1-4分) | 1个认知点 | 阅读/思考/观察 | 建立信心，体验小成功 |
| 中(5-8分) | 1认知+1小行动 | 思考+简单执行 | 完成基础改善 |
| 高(9-13分) | 1认知+2-3行动 | 完整执行任务 | 达成具体目标 |

#### 目标导向调整(目标类型)

| 目标类型 | 计划重心 | D7 验收标准 |
|---------|---------|------------|
| 止血型 | 解决最痛的点 | 紧急问题是否缓解 |
| 修复型 | 改善现有流程 | 流程是否优化 |
| 成长型 | 创造新可能 | 是否有新进展 |

## 五、追踪信设计

### 5.1 发信时机

| 时间点 | 信件类型 | 目的 |
|--------|---------|------|
| D0(提交后立即)| 完整报告信 | 交付价值、建立专业感 |
| D3 | 中途追踪信 | 确认进度、提供鼓励 |
| D7 | 成果回顾信 | 检视成果、导向下一步 |

### 5.2 追踪信个人化逻辑

```
D3 追踪信：

IF action_ability = "低" THEN
  语气：高度鼓励
  重点：「你已经踏出第一步了！」
  
IF predicted_obstacle = "太忙没时间" THEN
  加入：「只要今天花 5 分钟回顾一下...」
  
IF status_level = "红灯" THEN
  加入：「如果感到吃力，随时可以预约咨询...」
```

```
D7 追踪信：

IF goal_type = "止血" THEN
  问：「{goal_primary} 处理得如何了？」
  
IF goal_type = "成长" THEN
```
