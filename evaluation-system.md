## 二、計分公式

### 2.1 四大瓶颈分數計算

#### B1｜市場/需求瓶颈

```
B1_base = Q1 市場相关选项數 × 1.5

B1_chronic = Q7 中市場相关选项數 × 1.0
B1_acute = Q8 中市場相关选项數 × 1.5

B1_condition_bonus = IF(Q3 ≤ 3 AND Q4 ≤ 3) THEN 1 ELSE 0
// 決策不慢、執行不差，但还是缺客 → 更確定是市場問題

B1_score = B1_base + B1_chronic + B1_acute + B1_condition_bonus
```

**B1 相关选项**：
- Q1：新客來源不穩、成交率不如預期
- Q7：客户開发一直沒有穩定方法
- Q8：業績/訂單明顯下滑、重要客户流失或关係惡化

---

#### B2｜管理/決策瓶颈

```
B2_quantitative = (Q3 × 0.35) + (Q4 × 0.30) + (Q6 × 0.25) + (Q5 × 0.10)

B2_chronic = Q7 中管理相关选项數 × 1.0

B2_score = B2_quantitative + B2_chronic
```

**B2 相关选项**：
- Q1：決策常被拖慢、老闆负擔過重
- Q7：老闆是最大瓶颈、管理制度一直建不起來

---

#### B3｜執行/產能瓶颈

```
B3_quantitative = (Q4 × 0.45) + (Q5 × 0.30) + (Q6 × 0.25)

B3_chronic = Q7 中執行相关选项數 × 1.0
B3_acute = Q8 中執行相关选项數 × 1.5

B3_score = B3_quantitative + B3_chronic + B3_acute
```

**B3 相关选项**：
- Q1：團隊執行力不一致、人員留不住
- Q7：團隊執行力長期不穩定
- Q8：关鍵人員离職或鬧离職、某個專案/產品嚴重卡关

---

#### B4｜现金流瓶颈

```
B4_subjective = IF(Q1 包含「现金流壓力」) THEN 3 ELSE 0
             + IF(Q2 包含「现金流安全感」) THEN 2 ELSE 0

B4_chronic = IF(Q7 包含「现金流一直都偏緊」) THEN 1.5 ELSE 0
B4_acute = IF(Q8 包含「现金流突然吃緊」) THEN 2.0 ELSE 0

B4_score = B4_subjective + B4_chronic + B4_acute
```

---

### 2.2 主瓶颈判定邏輯

```
Step 1｜现金流優先检查
IF B4_score ≥ 4 THEN 主瓶颈 = B4，跳到結束

Step 2｜Boolean 條件检查
B1_valid = (Q1市場选项 ≥ 1) AND (Q3 ≤ 3) AND (Q4 ≤ 3)
B2_valid = (Q3 ≥ 4 AND Q4 ≥ 4) OR (Q6 ≥ 4)
B3_valid = (Q4 ≥ 4) AND (Q3 ≤ 3)
B4_valid = (Q1 包含「现金流壓力」) OR (Q2 包含「现金流安全感」)

Step 3｜從有效瓶颈中选最高分
valid_bottlenecks = 所有 Bx_valid = TRUE 的瓶颈
主瓶颈 = valid_bottlenecks 中 score 最高者

Step 4｜平手裁決(依存活優先序)
IF 多個瓶颈同分 THEN
  優先序：B4 > B2 > B3 > B1
  主瓶颈 = 優先序最高者

Step 5｜無有效瓶颈时
IF valid_bottlenecks 為空 THEN
  主瓶颈 = 所有瓶颈中 score 最高者
```

---

### 2.3 燈号判定邏輯

```
計算緊急度總分：

urgency_score = 
  主瓶颈分數
  + (Q8 勾选數 × 1.5)
  + IF(Q8 包含「现金流突然吃緊」) THEN 2 ELSE 0
  + IF(Q6 ≥ 4) THEN 1 ELSE 0

燈号判定：

🔴 紅燈條件(任一成立)：
  - urgency_score ≥ 7
  - B4_score ≥ 4
  - Q8 包含「现金流突然吃緊」
  - (Q8 勾选數 ≥ 2) AND (任一急性项目權重 ≥ 1.5)

🟡 黃燈條件(任一成立)：
  - 4 ≤ urgency_score < 7
  - 主瓶颈分數 ≥ 3
  - Q3 ≥ 4 OR Q4 ≥ 4 OR Q5 ≥ 4 OR Q6 ≥ 4
  - Q7 勾选數 ≥ 3

🟢 綠燈條件：
  - 以上都不符合
```

---

### 2.4 問題性質判定

```
acute_count = Q8 勾选數(排除「沒有，最近还算穩定」)
chronic_count = Q7 勾选數(排除「以上都沒有」)

問題性質判定：

「急性為主」：acute_count ≥ 2 AND chronic_count ≤ 1
「慢性為主」：chronic_count ≥ 3 AND acute_count ≤ 1
「複合型」：acute_count ≥ 1 AND chronic_count ≥ 2
「穩定型」：acute_count ≤ 1 AND chronic_count ≤ 1
```

---

### 2.5 行动能力計分

```
time_score = Q9 分數(1-4)
helper_score = MIN(Q10 分數加總, 3)(0-3，封頂)
confidence_score = Q14 分數 × 2(0-6)

action_ability_score = time_score + helper_score + confidence_score

範圍：1-13 分

行动能力分級：
  低(1-4 分)：認知建立，小成功体驗
  中(5-8 分)：認知 + 簡化行动  
  高(9-13 分)：完整實戰計畫
```

---

### 2.6 目标類型判定

```
根據 Q12 勾选项目：

IF 勾选任一「止血型」选项 THEN goal_type = "止血"
ELSE IF 勾选任一「修復型」选项 THEN goal_type = "修復"
ELSE IF 勾选任一「成長型」选项 THEN goal_type = "成長"

IF 勾选多個類型 THEN
  優先序：止血 > 修復 > 成長(先解決生存問題)
  但保留次要目标作為計畫的第二主軸
```

## 五、追蹤信設計

### 5.1 发信时機

| 时間点 | 信件類型 | 目的 |
|--------|---------|------|
| D0(提交后立即)| 完整報告信 | 交付價值、建立專業感 |
| D3 | 中途追蹤信 | 確認进度、提供鼓勵 |

### 5.2 追蹤信個人化邏輯

```
D3 追蹤信：

IF action_ability = "低" THEN
  語氣：高度鼓勵
  重点：「你已經踏出第一步了！」
  
IF predicted_obstacle = "太忙沒时間" THEN
```
